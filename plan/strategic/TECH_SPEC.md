# Reborn — 프론트엔드 기술 스펙

---

## 1. 기술 스택

| 영역 | 선택 |
|------|------|
| 프레임워크 | React |
| 언어 | TypeScript |
| 아이콘 | Lucide React |
| 스타일링 | Tailwind CSS (기본) + CSS Module (제한적 혼용) |

---

## 2. 스타일링 규칙

### 기본 원칙
- **Tailwind CSS**를 기본 스타일링 도구로 사용한다.
- **인라인 스타일은 원칙적으로 지양**한다.

### CSS Module 혼합 사용 허용 조건

다음 경우에 한하여 CSS Module 혼합 사용을 허용한다:

1. 컴포넌트 구조가 복잡하여 Tailwind className이 과도하게 길어지는 경우
2. 반복되는 스타일 묶음이 많아 가독성이 떨어지는 경우
3. 상태별 스타일, 하위 요소 스타일, 애니메이션 등 Tailwind만으로 표현 시 유지보수가 어려운 경우

---

## 3. 기타 규칙

| # | 규칙 | 설명 |
|---|------|------|
| 1 | Mock 데이터 분리 관리 | 별도 폴더(`/mock` 또는 `/data`)로 관리 후 import하여 사용 |
| 2 | 상태관리 라이브러리 사용 금지 | Redux, Zustand, Jotai 등 사용하지 않음. 상태관리는 개발단에서 별도 처리 예정 |

---

## 4. 프로토타입(기획서)과 프로덕션의 관계

> 프로토타입 코드는 프로덕션에서 **그대로 가져가서 사용**하므로, 동일한 기술 스택을 따른다.

| 영역 | 공통 스택 |
|------|----------|
| 프레임워크 | React |
| 언어 | TypeScript |
| 스타일 | Tailwind CSS + CSS Module (제한적) |
| 아이콘 | Lucide React |
| 차트 | ECharts + Recharts |
| 애니메이션 | Framer Motion + GSAP |
| 빌드 | Vite |

| 구분 | 프로토타입 (기획서) | 프로덕션 (개발) |
|------|-------------------|----------------|
| 데이터 | Mock 데이터 (하드코딩) | 실제 API 연동 |
| 상태관리 | 없음 | 개발단 결정 예정 |
| 배포 | Vercel | TBD |
| 라우팅 | react-router-dom | 동일 |
