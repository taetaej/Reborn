# Reborn — AIM (Audience Index Model) 기존 로직 스펙

> 기존 데이터브릭스 기반 AIM의 인풋/아웃풋 정리 + 솔루션 내재화 시 개선 방향

---

## 1. AIM 개요

| 항목 | 내용 |
|------|------|
| 정의 | 브랜드의 전반적인 변화 흐름을 종합적으로 평가하는 통합 지표 |
| 기존 환경 | 데이터브릭스 대시보드에서 추출 |
| 스코어링 | 0~100점, **50점 기준** |
| 판단 기준 | 50점 초과 = 긍정적 변화 / 50점 미만 = 부정적 변화 |
| 이번 목표 | 데이터브릭스에서 꺼내 Reborn 솔루션으로 내재화 + 고도화 |

---

## 2. 인풋 데이터 (Input)

### 2.1 Brand Awareness (브랜드 인지도)

| 항목 | 내용 |
|------|------|
| 데이터 소스 | 리스닝마인드 허블 |
| 의미 | 브랜드 검색량 기반 인지도 |
| 추출 방법 | 허블 '쿼리 파인더' → 키워드 입력 → 연관 키워드 선택 → 다운로드 |

**다운로드 결과 컬럼:**

| 컬럼 | 설명 |
|------|------|
| 키워드 | 검색어 |
| 월 평균 검색량 | 월간 평균 검색 볼륨 |
| 연간 총 검색량 | 12개월 합산 |
| 3개월 전 대비 증감률(%) | 최근 트렌드 |
| CPC (USD) | 클릭당 비용 |
| 광고경쟁도(%) | 광고 경쟁 수준 |
| 검색노출타입 | 검색 결과 노출 형태 |
| 검색 인텐트 | 정보/거래/탐색 의도 |
| 성별 특성 | 남성(%) / 여성(%) |
| 연령대별 특성 | 12세이하 ~ 50세이상 7구간 |
| 월별 검색량 | yyyy-mm 단위 시계열 (예: 2022-04 ~ 2026-03) |

---

### 2.2 Media Mentions & Media Positive Rate (언급량 + 감성)

| 항목 | 내용 |
|------|------|
| 데이터 소스 | 트렌드업 |
| 의미 | 광고/브랜드 언급량(버즈량) + 긍정/부정/중립 비율 기반 평판 |
| 추출 방법 | 트렌드업 '브랜드 분석' → 1년 월단위 → 키워드 입력 → 다운로드 |

**다운로드 결과 1: 매체별 버즈량 (4개 매체 각각)**

| 매체 | 데이터 형태 |
|------|------------|
| 매스미디어 | 월별 버즈량 시계열 |
| 인스타그램 + 유튜브 + 트위터 | 채널별 월별 버즈량 |
| 블로그 | 월별 버즈량 시계열 |
| 커뮤니티 | 월별 버즈량 시계열 |

**다운로드 결과 2: 긍/부정 추이 (4개 매체 각각)**

| 매체 | 데이터 형태 |
|------|------------|
| 매스미디어 | 월별 긍정/부정/중립 건수 |
| 인스타그램 + 유튜브 + 트위터 | 월별 긍정/부정/중립 건수 |
| 블로그 | 월별 긍정/부정/중립 건수 |
| 커뮤니티 | 월별 긍정/부정/중립 건수 |

총 **8개 파일** 다운로드 필요 (매체 4 × 지표 2)

---

### 2.3 Marketing Effect (마케팅 효과) — 선택

| 항목 | 내용 |
|------|------|
| 데이터 소스 | 코리안클릭 또는 GA |
| 필수 여부 | **선택** (없어도 AIM 산출 가능) |
| 의미 | 유입 기반 간접 성과 |

**(1) 코리안클릭**
- 조회조건: Mobile/PC/MO+PC, APP/WEB/AP+WEB, OS 선택
- 결과: 월별 순이용자 수 (MAU) 시계열

**(2) GA (Google Analytics)**
- 컬럼: 기준년월 / 세션 수
- 파일명: `{광고주명}.csv`
- ⚠️ AX팀 계정 정보 없음 → 광고주 측 제공 필요

### 2.4 DBX 추출 시 입력 조건 (기존)

| 단계 | 입력 항목 |
|------|----------|
| 1 | 파일 경로 설정 + 다운로드 파일 업로드 |
| 2 | 캠페인 목적 선택 |
| 3 | 키워드 입력 |
| 4 | 분석 기간 선택 |
| 5 | 광고주(브랜드), 경쟁사 선택 |

## 3. 아웃풋 데이터 (Output)

### 3.1 개별 지표 (Raw Data)

| 지표 | 소스 | 의미 |
|------|------|------|
| Brand Awareness | 리스닝마인드 허블 | 브랜드 검색량 기반 인지도 |
| Media Mentions | 트렌드업 | 광고/브랜드 언급량 (버즈량) |
| Media Positive Rate | 트렌드업 | 긍/부정/중립 비율 기반 평판 |
| Marketing Effect | GA / 코리안클릭 | 유입 기반 간접 성과 |

### 3.2 통합 지표 (AIM Score)

| 지표 | 정의 | 특징 | 활용 |
|------|------|------|------|
| **종합 AIM** | 단기+중기+장기 흐름을 함께 반영한 종합 평가 | 브랜드 전체 방향성 파악 | 브랜드 성장/하락 추이, 장기 의사결정 |
| **단기 AIM** | 전월 대비 변화 중심으로 즉각적 효과 평가 | 특정 시점 변화에 민감 | 캠페인 직후 효과 판단, 전략 수정 |

### 3.3 스코어링 체계

```
0 ─────────── 50 ─────────── 100
   부정적 변화    기준점     긍정적 변화
   (악화)        (변화없음)   (개선)
```

- **50점 초과**: 전반적으로 긍정적 변화. 점수가 높을수록 개선 정도 큼
- **50점 미만**: 전반적으로 부정적 변화. 점수가 낮을수록 악화 정도 큼
- **50점**: 유의미한 변화 없음 (베이스라인)

## 4. 기존 워크플로우 (AS-IS)

```
사용자(기획자/AM)
  │
  ├─ 리스닝마인드 허블 로그인 → 쿼리 파인더 → 키워드 입력 → 다운로드
  ├─ 트렌드업 로그인 → 브랜드 분석 → 8개 파일 다운로드
  ├─ (선택) 코리안클릭/GA → 월별 MAU/세션 수 다운로드
  │
  ▼
데이터브릭스
  ├─ 파일 업로드
  ├─ 캠페인 목적/키워드/기간/브랜드 입력
  │
  ▼
AIM 결과 출력
  ├─ 종합 AIM 점수
  ├─ 단기 AIM 점수
  └─ 개별 지표 시계열
```

**기존 문제점:**
1. 사용자가 3개 플랫폼에 각각 로그인해야 함
2. 8~10개 파일을 수동 다운로드
3. 데이터브릭스에 다시 업로드
4. 전체 프로세스 소요 시간: 30분~1시간+
5. 실수 가능성 (파일 누락, 기간 불일치 등)

## 5. Reborn 내재화 후 워크플로우 (TO-BE)

```
사용자
  │
  ├─ Thread(워크스페이스) 진입
  ├─ 대화형: "삼성전자 최근 1년 AIM 보고 싶어"
  │   또는 수동: 키워드+기간+브랜드 직접 입력
  │
  ▼
Reborn 백엔드 (자동)
  ├─ Fiber "검색량" ← 네이버 API + 리스닝마인드 크롤링
  ├─ Fiber "소셜 버즈" + "감성 신호" ← 트렌드업 크롤링
  ├─ Fiber "마케팅 유입" ← GA 연동 or 업로드 (선택)
  │
  ▼
AIM Loom 실행
  ├─ 데이터 정규화
  ├─ AIM 모델 연산
  │
  ▼
Ribbon (산출물) 생성
  ├─ 종합 AIM 점수 + 단기 AIM 점수
  ├─ 월별 시계열 차트
  ├─ 개별 지표 드릴다운
  └─ HEXIM 자연어 해석 ("전월 대비 3점 상승, 주요 원인은...")
```

**개선 효과:**
| AS-IS | TO-BE |
|-------|-------|
| 3개 플랫폼 로그인 | Reborn 하나에서 완결 |
| 8~10개 파일 수동 다운로드 | 자동 수집 (Fiber) |
| 데이터브릭스 업로드 | 불필요 |
| 소요 시간 30분~1시간 | **수 분 이내** |
| 결과 = 숫자만 | 결과 = 숫자 + 차트 + **자연어 해석** |

## 6. 고도화 방향 (민지님 싱크 후 확정 예정)

| # | 고도화 항목 | 설명 |
|---|------------|------|
| 1 | AIM 연산 모델 개선 | 기존 가중치 조정, 새 변수 추가 검토 |
| 2 | X 언급 데이터 추가 (via Grok) | 기존에 없던 X/트위터 데이터를 Grok API로 보충 |
| 3 | 경쟁사 비교 AIM | 자사 AIM vs 경쟁사 AIM 자동 비교 (Phase 2) |
| 4 | HEXIM 해석 자동화 | 점수 변동 원인을 자연어로 자동 설명 |
| 5 | 알림 체계 | AIM 급변 시 자동 알림 (Thread에 설정) |
| 6 | 시계열 예측 | 향후 AIM 추이 예측 (ML 모델, Phase 2) |

## 7. Fiber ↔ AIM 인풋 맵핑

| AIM 인풋 | 기존 소스 | Reborn Fiber | 수집 방식 |
|----------|----------|-------------|----------|
| Brand Awareness | 리스닝마인드 허블 | 검색량 + 연관 검색어 | 크롤링 (또는 네이버 API 대체 가능) |
| Media Mentions | 트렌드업 (매체별 버즈량) | 소셜 버즈 | 크롤링 |
| Media Positive Rate | 트렌드업 (긍/부정 추이) | 감성 신호 | 크롤링 |
| Marketing Effect | 코리안클릭 / GA | 마케팅 유입 (신규 Fiber) | 업로드 or GA 연동 |

### 참고: 네이버 API로 Brand Awareness 일부 대체 가능

| 허블 데이터 | 네이버 API 대체 가능 여부 |
|------------|:---:|
| 월별 검색량 시계열 | ✅ 네이버 데이터랩 API (상대값) + 검색광고 API (절대값) |
| 3개월 전 대비 증감률 | ✅ 계산 가능 |
| 성별/연령대별 특성 | ✅ 네이버 데이터랩 API 제공 |
| 검색 인텐트 | ❌ 허블 고유 데이터 |
| CPC/광고경쟁도 | ✅ 검색광고 API 제공 |
| 연관 키워드 목록 | ✅ 검색광고 API 제공 |

→ **Brand Awareness의 대부분은 네이버 API로 대체 가능**. 허블 고유 가치는 "검색 인텐트"와 "패스파인더(검색 여정)" 데이터.
