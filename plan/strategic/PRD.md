# Reborn — Product Requirements Document

> **"Read the 본심, Reborn to Insight"**
> 캠페인 집행 후 오디언스의 진심을 읽고, 브랜드 인사이트로 재탄생시키는 플랫폼

---

## 1. Executive Summary

### 배경
디지털 마케팅의 정량 지표(노출, 클릭 등)만으로는 "우리 브랜드가 얼마나 좋아졌나?"라는 광고주의 본질적 질문과 의사결정을 지원할 수 없음.

### 제품 정의
Reborn은 ADLY 생태계 내에서 파편화된 정성·정량 신호로부터 광고주를 설득할 수 있는 **가장 강력한 마케팅 내러티브를 자동으로 추출해 주는 엔진**이다. 여러 데이터를 꿰어서 내러티브를 발굴하고, 그 결과를 즉각적인 의사결정이 가능한 '해석'으로 전환하는 브랜드·오디언스 인텔리전스 플랫폼.

### 핵심 구조
```
데이터소스(Sources) → 도구(Tools) → 산출물(Outputs)
```
이 구조가 **워크스페이스** 단위로 격리되며, 워크스페이스는 **광고주**에 1:1 맵핑됨.

### 당해년도 과제
1. 멀티 솔루션을 수용하는 확장형 플랫폼 구조(파이프라인) 확립
2. 첫 번째 핵심 솔루션인 **AIM(Audience Index Model)** 개선 및 내재화
3. HEXIM 엔진 기반 자연어 해석 레이어 MVP 런칭

## 2. 포지션 및 관계

| 항목 | ReadySet | Reborn |
|------|----------|--------|
| 성격 | 캠페인 전략 플랫폼 (사전) | 오디언스 인사이트 플랫폼 (사후) |
| 핵심 질문 | "누구에게, 어떻게 도달할까?" | "우리 브랜드가 어떻게 인식되고 있나?" |
| 단위 개념 | 슬롯 (Slot) | 워크스페이스 (Workspace) |
| 주요 도구 | 리치캐스터, 데이터셋 | AIM, 소셜 클러스터링 등 |
| 관계 | 형제 플랫폼 — GNB, 폰트 패밀리, 컴포넌트 체계 공유 |

**인사이트 피드백 루프**: ReadySet에서 수립한 타겟 가설 → Reborn에서 실제 오디언스 반응 검증 → 차기 캠페인 전략에 환류

## 3. 목적 및 목표

### 3.1 전략적 목표
- **인사이트 피드백 루프 완성**: 캠페인 기획 단계의 타겟 가설과 실제 집행 중 오디언스 반응 사이의 간극을 입증하고 차기 전략에 환류
- **설명 가능한 의사결정 지원**: 광고주에게 "다음 분기에 어디에 얼마를 쓸 것인가"에 대한 명확한 전략적 근거 제공

### 3.2 기술적 목표
- **멀티 솔루션 파이프라인 아키텍처** (최우선): [데이터 수집 → 정규화 → 모델링 → 점수화 → 해석 → 리포트]의 공통 흐름을 아키텍처화하여 향후 솔루션이 플러그인될 수 있는 기반 확립
- **데이터 인게스트 자동화**: 외부 소스(트렌드업 등)를 API 기반으로 자동 수집하는 파이프라인 구축
- 운영 편의를 위한 백엔드 스케줄러 기반 정기 업데이트 지원

### 3.3 비즈니스 목표
- **사내 분석 자산화**: 파편화되어 사라지던 브랜드 정성 데이터를 플랫폼 내 축적
- **전사적 브랜드 리터러시 확보**: 일부 전문가 의존에서 벗어나 누구나 인사이트에 접근 가능

## 4. 시장 배경 및 기회 요인

### 4.1 시장 흐름
- **정량 지표의 한계**: 노출, 클릭, 한정적 전환 수치만으로는 브랜드 팬덤이나 진정성 있는 연결을 증명할 수 없음
- **의사결정 영역의 공백**: 광고주의 실제 핵심 의사결정(예산 재분배 등)은 '설명되지 않는 영역'에서 결정됨 → 정성 신호 인사이트 솔루션이 미디어렙사의 핵심 경쟁력으로 부상

### 4.2 사용자 문제 (User Problems)
| 문제 | 설명 |
|------|------|
| 해석의 파편화 | 방대한 텍스트 데이터를 통합 해석할 도구 부재 → 기획자 주관적 직관 의존 |
| 증명 불가한 정성 성과 | 캠페인이 인식을 어떻게 변화시켰는지 입증할 객관적 지표 부재 |
| 분석 인프라 접근성 저하 | 수동 가공에 과도한 리소스 투입 → 실무자 공수 매몰 |

### 4.3 기회 요인
- **인사이트의 자동 자산화 (AIM)**: 검색량, 미디어 언급, 감성, 유입 등 신호를 자동 수집·누적 관리
- **해석 솔루션 포지셔닝**: HEXIM 엔진을 통한 자연어 해석 레이어 → 분석가 없이도 즉각 보고서 활용 가능한 고품질 인사이트 제공

## 5. 거버넌스 및 데이터 격리

### 5.1 사용자 롤 체계

| Role | 설명 |
|------|------|
| Admin | 시스템 전체 관리, 광고주 생성/삭제, 권한 배정 |
| Marketer | 내부 마케터, 워크스페이스 생성 및 도구 실행 |
| Agency | 외부 에이전시, 할당된 광고주 워크스페이스 접근 |
| Client | 광고주 본인, 산출물 열람 및 공유 |

### 5.2 데이터 격리 구조

```
User (4 Roles)
  └─ has many ─→ 광고주 권한 (삼성전자, LG전자, …)
                    └─ maps to ─→ Workspace(s)
```

- **격리 기준** = 광고주 단위
- 1 유저 → N개 광고주 권한 가능
- 유저에게 보이는 워크스페이스 = 본인이 권한을 가진 광고주에 맵핑된 것만
- ReadySet의 "슬롯(Slot)" 개념과 동일한 격리 목적

## 6. 플랫폼 구조: 데이터소스 → 도구 → 산출물

### 6.1 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────┐
│                   WORKSPACE (광고주 단위)                      │
├──────────────┬───────────────────┬──────────────────────────┤
│  데이터소스     │       도구         │        산출물             │
├──────────────┼───────────────────┼──────────────────────────┤
│ 트렌드업       │  AIM              │ 월별 시계열 차트 + 해석    │
│ (버즈량, TPO,  │  (브랜드 지표)      │                          │
│  감성, 원문)   │                   │                          │
├──────────────┤                   ├──────────────────────────┤
│ 리스닝마인드    │  소셜 클러스터링    │ 반응 클러스터 리포트       │
│ 허블          │                   │                          │
├──────────────┤                   ├──────────────────────────┤
│ Grok         │  (Phase 2 확장)    │                          │
│ (X/트위터)    │  영상분석           │ 영상 인사이트 리포트       │
├──────────────┤  경쟁사 비교        │ 비교 대시보드             │
│ 캠페인 브리프  │                   │                          │
├──────────────┤                   │                          │
│ 웹 검색/참고  │                   │                          │
│ (그라운딩)    │                   │                          │
└──────────────┴───────────────────┴──────────────────────────┘
```

### 6.2 설계 원칙
1. **파이프라인 우선**: 건물(구조)을 먼저 확립하고, 도구·산출물은 플러그인으로 확장
2. **데이터 충족 검증**: 도구 실행 전 "이 도구에 필요한 데이터소스가 충분한가?" 사전 검증
3. **해석 레이어 기본 탑재**: 모든 산출물에 HEXIM 기반 자연어 해석 포함

## 7. 데이터소스 상세

### 7.1 Fiber 추상화 원칙

사용자에게는 **실제 소스명(트렌드업, 리스닝마인드 등)을 노출하지 않는다.** 데이터 유형(Fiber 타입)만 보여주고, 뒷단에서 어떤 서비스로 수집하는지는 추상화한다.

> 이점: 소스가 교체되어도 사용자 경험 변화 없음. UI에서 특정 서비스 의존 제거.

### 7.2 Fiber 유형 → 실제 소스 맵핑

| Fiber (사용자에게 보이는 이름) | 설명 | 실제 소스 (백엔드) | 수집 방식 |
|------|------|------|------|
| **소셜 버즈 원문** | 소셜 미디어 실제 사용자 발화 원문 | 트렌드업 | 키워드+기간 기반 크롤링 |
| **감성 신호** | 긍정/부정/중립 감성 분포 | 트렌드업 | 키워드+기간 기반 크롤링 |
| **TPO 키워드** | 시간/장소/상황 기반 연관 키워드 | 트렌드업 | 키워드+기간 기반 크롤링 |
| **검색량** | 키워드 검색 볼륨 추이 | 네이버 데이터랩 / 구글 트렌드 | API 또는 크롤링 |
| **연관 검색어** | 키워드와 함께 검색되는 연관어 | 리스닝마인드 허블 / 네이버 | 크롤링 |
| **검색 의도/여정** | 검색 기반 소비자 관심사·니즈 흐름 | 리스닝마인드 허블 | 크롤링 |
| **캠페인 브리프** | 기획 맥락, 타겟 가설 | 수동 업로드 or ReadySet 연동 | 업로드 |
| **외부 레퍼런스** | 뉴스, 아티클, 업계 동향 | 웹 검색 | 자동 수집 |

### 7.3 Fiber 수집 플로우 (사용자 관점)

```
① Fiber 유형 선택 (예: "소셜 버즈 원문")
② 키워드 입력 (예: "갤럭시 S25")
③ 기간 설정 (예: "2026.05.01 ~ 2026.05.31")
④ [수집 시작] → 백엔드가 해당 소스에서 크롤링
⑤ 완료 시 Thread에 Fiber로 저장 → Loom(도구)에서 활용 가능
```

> 대화형 UX에서는 ①~③을 챗으로 자동 추출 가능:
> "이번 달 갤럭시 S25 소셜 반응 보고 싶어" → AI가 키워드·기간·Fiber 유형 자동 설정

### 7.4 데이터 인게스트 방식 및 제약사항

| 소스 | 현재 방식 | 목표 방식 | 제약/미확정 |
|------|----------|----------|------------|
| 트렌드업 | 사용자가 직접 다운로드 → 업로드 | 키워드+기간 기반 자동 크롤링 | ⚠️ 법적 검토 필요 (아래 참조) |
| 리스닝마인드 허블 | 사용자가 직접 다운로드 → 업로드 | 자동 크롤링 | ⚠️ 법적 검토 필요 |
| 네이버 데이터랩 | - | API 연동 (공개 API 존재) | Rate limit 확인 필요 |
| 구글 트렌드 | - | 비공식 크롤링 or pytrends | 안정성 이슈 |
| 웹 검색 | - | 자동 수집 | 그라운딩 용도 |

### 7.5 크롤링 법적 검토 필요 사항

트렌드업·리스닝마인드 허블 모두 유료 이용 중이나, 크롤링 자동화에 대해 아래 확인 필요:

| 확인 항목 | 설명 |
|----------|------|
| 이용약관(ToS) 크롤링 금지 조항 | 대부분 SaaS에 "자동화된 수단 접근 금지" 조항 존재 가능 |
| 데이터 이용 범위 제한 | "본 서비스 내에서만 열람" 조항 여부 |
| 개인정보 포함 | 소셜 원문에 닉네임·프로필 등 포함 시 개인정보보호법 적용 |

**권장 조치**:
1. 양사 담당자에게 "크롤링 자동 수집 가능 여부" 서면 확인 요청
2. 또는 데이터 연동 제휴/API 제공 요청 (유료 고객으로서 협상력 있음)
3. 트렌드업은 API 문의 이력 있으나 비용 과다 → 크롤링 허용 여부 협의 우선

> ⚠️ 상태: 법적 검토 미완료. 기획 단계에서 구조는 크롤링 기준으로 설계하되, 실제 운영 전 서면 동의 확보 필수.

## 8. 솔루션 모듈 (도구)

### 8.1 AIM (Audience Index Model) — Phase 1

**개요**: 브랜드의 오디언스 반응을 종합적으로 측정하는 평판 지표

**스코어링 체계**:
- 기준월 = 50점 (베이스라인)
- 50 이상 = 브랜드 상태 개선
- 50 이하 = 브랜드 상태 하락

**입력 데이터**: 트렌드업 버즈량 + 리스닝마인드 데이터 + 감성 분석 등

**산출물**:
- 월별 시계열 점수 차트
- HEXIM 엔진 기반 자연어 해석 (점수 변동 원인 설명)
- 주요 드라이버 분석

> ⚠️ TO BE UPDATED — 민지님 싱크 후 AIM 개선 모델 연산 로직 확정 예정

### 8.2 소셜 반응 클러스터링 — Phase 1

**개요**: 소셜 언급 원문을 LLM 기반으로 분류·그룹핑하여 오디언스 반응 유형 파악

**입력 데이터**: 트렌드업 소셜 언급 원문

**산출물**:
- 클러스터별 키워드 그룹 시각화
- 원문 샘플 (대표 멘션)
- 클러스터별 요약 및 인사이트

### 8.3 Phase 2 확장 예정
- 영상 분석 (해커톤 원형 활용)
- 경쟁 브랜드 비교
- 캠페인 사전·사후 효과 측정

## 10. UX 방향 및 벤치마크

### 10.1 컨셉 방향성
데이터소스 ↔ 도구 ↔ 결과 캔버스가 유기적으로 연결되는 **전문가 워크스페이스** 형태. LLM이 개입하는 분석 프로세스를 투명하고 직관적으로 표현.

### 10.2 벤치마크 서비스 분석

#### 1) Google NotebookLM — "소스 기반 AI 리서치 워크스테이션"
- **참고 포인트**: 소스 업로드 → AI가 소스 기반으로 답변·요약·인포그래픽 자동 생성. 2026.6 업데이트로 멀티 스텝 리서치 에이전트 + 코드 실행 + 산출물 내보내기까지 확장됨
- **Reborn 적용**: 데이터소스를 올리면 AI가 해당 소스를 기반으로 분석 결과를 생성하는 흐름의 UX 레퍼런스. 소스 신뢰성 표시(인용 연결) 방식 참고
- [notebooklm.google](https://notebooklm.google)

#### 2) Hex — "AI 데이터 노트북 + 인터랙티브 앱"
- **참고 포인트**: SQL/Python 셀 + 차트 빌더 + 마크다운이 하나의 캔버스에 공존. Notebook Agent가 비즈니스 질문 → 분석 계획 수립 → 코드 실행 → 결과 요약을 자동 수행. 완성된 노트북은 인터랙티브 앱으로 퍼블리시 가능
- **Reborn 적용**: "도구 실행 → 결과 캔버스" 구조의 UX 레퍼런스. 비기술 사용자도 결과를 인터랙티브하게 탐색할 수 있는 앱 형태 참고
- [hex.tech](https://hex.tech)

#### 3) Julius AI — "대화형 데이터 분석 워크스페이스"
- **참고 포인트**: 데이터소스 연결(Google Ads, Snowflake, Excel 등) → 자연어로 분석 요청 → 차트/리포트 자동 생성. 팀 워크스페이스 개념으로 협업 지원
- **Reborn 적용**: 비분석가가 자연어로 인사이트를 요청하는 UX 패턴. 멀티 데이터소스 연결 시 시각적 피드백 방식 참고
- [julius.ai](https://julius.ai)

#### 4) Akkio — "광고 에이전시 특화 AI 분석 플랫폼"
- **참고 포인트**: 광고 에이전시가 핵심 타겟. 캠페인 데이터를 자연어로 질의하여 클라이언트용 리포트 생성. 기존 데이터 권한을 그대로 계승하는 격리 구조
- **Reborn 적용**: 광고주 데이터 격리 + 에이전시 워크플로우에 특화된 UX. "Chat Engine"으로 데이터에 질의하는 패턴이 HEXIM 해석 레이어와 유사
- [akkio.com](https://akkio.com)

#### 5) Pulsar Platform — "오디언스 인텔리전스 + 내러티브 AI" ⭐ 핵심 벤치마크

**제품 스택 구조**:
```
Workspaces (조직 계층, 팀/클라이언트 격리)
  ├─ TRAC (리스닝 + 커뮤니티 감지, 45+ 소스, 200+ 언어)
  ├─ Narratives AI (내러티브 클러스터링 — 키워드 빈도 X, '이야기' 단위)
  ├─ CORE (오운드채널 분석)
  └─ Saga (자율 리서치 에이전트 — 브리프 한 번 → 24/7 자동 모니터링)
```

**Pulsar의 검증된 패턴 중 Reborn에 차용할 것**:

| # | Pulsar 패턴 | Reborn 적용 방향 |
|---|------------|-----------------|
| 1 | **커뮤니티 감지** — 키워드 수가 아닌 "누가 말하는지" 네트워크 기반 그룹핑 | 소셜 클러스터링 도구에서 단순 토픽 분류 X → 발화자 커뮤니티 기반 세그먼트 구현 |
| 2 | **Narratives AI** — 개별 멘션이 아닌 '내러티브(이야기)' 단위로 대화 묶기 | HEXIM 해석에 "이 기간에 형성된 주요 내러티브 3개와 각각의 감성 변화" 패턴 적용 |
| 3 | **Saga 자율 에이전트** — 브리프 1회 → 에이전트가 24/7 모니터링 후 결과 푸시 | 대화형 UX에서 "처음에 설정 → 이후 자동 반복 모니터링" 기능으로 발전. "살아 움직이는 플랫폼" 비전에 부합 |
| 4 | **Workspaces + Search Folders** — 워크스페이스 안에 폴더로 검색/분석 구조화 | 워크스페이스 안에서 산출물·도구 실행 이력을 폴더/탭 기반으로 정리하는 UX 차용 |
| 5 | **커스텀 대시보드** — 50+ 시각화에서 원하는 차트를 골라 조합 | 산출물 캔버스를 위젯 조합 방식으로 (AIM 차트 + 클러스터 맵 + 해석 텍스트 자유 배치) |
| 6 | **Visibility 메트릭** — 단순 볼륨 아닌 임팩트 점수 (노출+인게이지먼트+채널 가중) | AIM 점수 설계 시 복합 가중 합산 로직의 레퍼런스 |

**Reborn이 Pulsar보다 잘할 수 있는 지점**:

| 차별점 | 설명 |
|--------|------|
| **AIM 단일 지표** | Pulsar는 풍부한 데이터를 주지만 "좋아졌어? 나빠졌어?"를 한 숫자로 안 알려줌. AIM 50점 기준은 의사결정을 극적으로 단순화 |
| **자연어 해석 디폴트** | Pulsar는 시각화를 던져주고 해석은 사용자 몫. Reborn은 "왜?"를 문장으로 설명하는 게 기본값 → 비전문가 접근성 압도 |
| **캠페인 생태계 연동** | ReadySet↔Reborn으로 "전략→집행→검증→환류"가 하나의 생태계에서 완결. Pulsar에는 이 루프 없음 |
| **대화형 UX** | Pulsar는 Boolean 쿼리(5000자)로 검색 생성. 진입 장벽 높음. Reborn의 "대화로 시작" 접근은 비전문가 온보딩에서 완승 |

- [pulsarplatform.com](https://www.pulsarplatform.com)

#### 6) Gamma — "AI 네이티브 프레젠테이션 + 캔버스"
- **참고 포인트**: 텍스트/데이터 입력 → AI가 구조화된 시각 산출물(프레젠테이션, 인포그래픽) 자동 생성. "Imagine" 캔버스로 자유형 비주얼 생성

- [gamma.app](https://gamma.app)

#### 7) Brandwatch — "소셜 리스닝 대시보드의 산업 표준"
- **참고 포인트**: 1.6T 히스토리컬 대화 데이터, Iris AI로 실시간 인사이트. Summary Dashboard로 핵심 지표를 한눈에 파악. Football Attention Index 같은 실시간 이벤트 분석 사례
- **Reborn 적용**: 실시간 트렌딩 위젯 + 시계열 대시보드의 비주얼 레퍼런스. 홈 화면의 "살아 움직이는" 느낌 구현 시 참고
- [brandwatch.com](https://www.brandwatch.com)

#### 8) Dovetail — "고객 인텔리전스 플랫폼"
- **참고 포인트**: 다양한 정성 데이터(인터뷰, 설문, 소셜 코멘트)를 한 곳에 모아 태깅·테마화·검색. AI 기반 자동 분류 + 테마 연결 + 비즈니스 임팩트 시각화
- **Reborn 적용**: 정성 데이터의 체계적 저장·분류·검색 UX. 데이터소스 수집 후 자동 분류되는 흐름의 레퍼런스
- [dovetail.com](https://dovetail.com)

### 10.3 UX 핵심 과제
- 데이터소스 → 도구 → 산출물의 3단계를 **어떤 레이아웃**으로 표현할 것인가 (3패널 동시 vs 스텝 vs 하이브리드)
- LLM 분석 과정의 투명성 (로딩 상태, 중간 단계 시각화)
- 도구 선택 시 필요 데이터소스 충족 여부 시각적 안내
- AX팀 퀄리티에 맞는 인터랙션·모션 디자인
