## 1. 네이버 데이터랩 (통합 검색어 트렌드 API)

| 항목 | 내용 |
|------|------|
| 제공사 | 네이버 |
| API 유형 | REST API (네이버 Open API) |
| 비용 | **무료** |
| 인증 | 네이버 개발자센터 앱 등록 → Client ID + Secret |
| Rate Limit | 일 1,000회 호출 (제휴 신청 시 상향 가능) |
| 제공 데이터 | 검색어 상대 검색량 추이 (1~100 스케일), 기간별/성별/연령별 |
| 기간 범위 | 2016.01.01 ~ 현재 |
| 응답 형식 | JSON |
| 연결 난이도 | ⭐ (매우 쉬움) |

**결론: ✅ 즉시 연동 가능. 무료. Fiber "검색량"에 직접 활용.**

## 2. 네이버 검색광고 API (연관 키워드 + 월간 검색량)

| 항목 | 내용 |
|------|------|
| 제공사 | 네이버 검색광고 |
| API 유형 | REST API |
| 비용 | **무료** (검색광고 계정만 있으면 됨, 광고비 집행 불필요) |
| 인증 | 검색광고 계정 → API 라이선스 발급 → API Key + Secret + Customer ID |
| Rate Limit | 분당 약 100회 |
| 제공 데이터 | 연관 키워드 목록, **월간 검색량 (절대값)**, PC/모바일 비율, 경쟁 정도 |
| 응답 형식 | JSON |
| 연결 난이도 | ⭐⭐ (쉬움, 인증 절차만 있음) |

**결론: ✅ 즉시 연동 가능. 무료. 네이버 데이터랩과 달리 절대 검색량 제공 → AIM 연산에 핵심.**

### 참고: 네이버 연관검색어 서비스 종료 (2025.04)
- 네이버가 연관검색어 서비스를 2025.04.30 종료함
- 그러나 **검색광고 API의 "연관 키워드" 기능은 별개**이며 계속 사용 가능
- 리스닝마인드 허블의 연관 키워드 데이터로 보완 가능

## 3. 구글 트렌드 API (alpha)

| 항목 | 내용 |
|------|------|
| 제공사 | Google |
| API 유형 | REST API (2025.07 런칭, alpha) |
| 비용 | **무료 (예상)** — alpha 단계이므로 향후 변동 가능 |
| 접근 조건 | **알파 테스터 신청 필요** (매우 제한적 인원만 승인) |
| 제공 데이터 | 검색 관심도 (일/주/월/년 집계), 지역별, 5년 히스토리컬 |
| 응답 형식 | JSON |
| 연결 난이도 | ⭐⭐⭐ (승인 받기가 어려움) |

### 대안: pytrends (비공식 Python 라이브러리)

| 항목 | 내용 |
|------|------|
| 비용 | 무료 (오픈소스) |
| 안정성 | ⚠️ 불안정 — Google이 비공식 접근 차단 가능 |
| 제공 데이터 | Interest over time, Related queries, Related topics, Interest by region |
| 리스크 | IP 차단, CAPTCHA, 불규칙 응답 |
| 연결 난이도 | ⭐⭐ (코드 쉽지만 운영 불안정) |

### 대안 2: SerpAPI / Bright Data 등 서드파티

| 항목 | 내용 |
|------|------|
| 비용 | 유료 ($50~200/월 수준) |
| 안정성 | ✅ 안정 |
| 설명 | 구글 트렌드 페이지를 대신 스크래핑해서 JSON으로 제공 |

**결론: ⚠️ 공식 API는 alpha 신청 필요. 단기적으로는 pytrends 또는 서드파티 활용, 장기적으로 공식 API 전환.**

## 4. Grok API (X/트위터 데이터)

| 항목 | 내용 |
|------|------|
| 제공사 | xAI |
| API 유형 | REST API (OpenAI 호환 형식) |
| 비용 | **유료 (종량제)** |
| 가격 | Grok 4.1 Fast: 입력 $0.20/MTok, 출력 $0.50/MTok |
| 무료 크레딧 | 신규 가입 시 **$25 무료**, 데이터 공유 프로그램 참여 시 $150/월 추가 |
| 인증 | xAI 계정 → API Key 발급 |
| 핵심 기능 | **Real-time web and X search** (빌트인 도구) |
| X 데이터 접근 | Grok이 X(트위터) 데이터에 실시간 접근 → LLM 응답 형태로 추출 |
| Rate Limit | Tier 기반 (무료 $0 → $50 → $250 → $1000 → $5000 순으로 상향) |
| 응답 형식 | JSON (OpenAI chat completions 호환) |
| 연결 난이도 | ⭐⭐ (OpenAI SDK 호환이라 쉬움) |

### X 데이터 접근 방식
- Grok API의 **빌트인 "live search" 도구** 활용
- 프롬프트로 "갤럭시 S25에 대한 최근 X 반응을 요약해줘" → X 데이터 기반 응답
- 직접 크롤링 아님, LLM이 X 데이터를 읽고 요약/분석하는 형태
- **정량 정확도 한계**: 정확한 멘션 수보다는 요약·감성·주요 의견에 강점

### 비용 시뮬레이션 (예시)
- AIM 1회 실행 시 Grok 호출: 약 2,000 입력 토큰 + 1,000 출력 토큰
- 비용: ~$0.0009/회 (1원 이하)
- 월 100회 실행 가정: ~$0.09 (약 120원)
- → **비용 거의 무시할 수준**

**결론: ✅ 즉시 연동 가능. 비용 매우 저렴. X 실시간 데이터를 LLM 응답으로 받는 형태.**

## 5. 트렌드업

| 항목 | 내용 |
|------|------|
| 제공사 | 트렌드업 (국내) |
| 공식 API | 존재하나 **비용 과다** (문의 이력 있음) |
| 현재 이용 상태 | ✅ 유료 구독 중 |
| 목표 연결 방식 | 웹 크롤링 (로그인 세션 유지 → 조건 입력 → 결과 수집) |
| 비용 | 추가 비용 없음 (이미 구독료 납부 중) |
| 리스크 | ⚠️ ToS 위반 가능성 → 서면 동의 필요 |
| 제공 데이터 | 버즈량, 소셜 원문, 감성 분석, TPO 키워드, 채널별 분포, 연관어 |
| 연결 난이도 | ⭐⭐⭐⭐ (크롤링 개발 + 로그인 세션 관리 + 안정성 유지) |

### 필요 액션
1. 트렌드업 담당자에게 "자동 수집 가능 여부" 서면 확인
2. 또는 API 비용 재협상 (데이터 제한적 범위만 활용 시 할인 가능한지)
3. 크롤링 허용 시: Puppeteer/Playwright 기반 헤드리스 브라우저 크롤러 개발

**결론: ⚠️ 기술적으로 가능하나 법적 확인 선행 필요. API는 비용 과다.**

## 6. 리스닝마인드 허블

| 항목 | 내용 |
|------|------|
| 제공사 | 어센트코리아 |
| 공식 API | 미확인 (별도 문의 필요) |
| 현재 이용 상태 | ✅ 유료 구독 중 |
| 목표 연결 방식 | 웹 크롤링 또는 API (있다면) |
| 비용 | 추가 비용 없음 (이미 구독료 납부 중) |
| 리스크 | ⚠️ ToS 위반 가능성 → 서면 동의 필요 |
| 제공 데이터 | 검색량, 인텐트 분류, 패스파인더(검색 여정), 클러스터, 연관어 맵 |
| 연결 난이도 | ⭐⭐⭐⭐ (크롤링 개발 + SPA 구조 파싱 복잡 가능성) |

### 필요 액션
1. 어센트코리아에 API 제공 여부 문의
2. API 없을 시: 크롤링 허용 여부 서면 확인
3. 크롤링 시: SPA(React 등) 구조라면 Playwright 필수

**결론: ⚠️ API 존재 여부부터 확인 필요. 없으면 크롤링 + 법적 확인.**
