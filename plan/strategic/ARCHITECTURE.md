# Reborn — 솔루션 아키텍처


---

## 1. 핵심 구조: 직조 파이프라인

Reborn은 **"Fiber → Loom → Ribbon"** 의 직조 파이프라인을 기본 골격으로 한다.

```
┌─────────────────────────────────────────────────────────────┐
│                    Thread (워크스페이스)                       │
│                    = 광고주 단위 작업 공간                     │
│                                                             │
│   ┌──────────┐      ┌──────────┐      ┌──────────────┐     │
│   │  Fiber   │ ───▶ │   Loom   │ ───▶ │   Ribbon     │     │
│   │ (원료)   │      │ (베틀)   │      │ (직조된 산출물) │     │
│   └──────────┘      └──────────┘      └──────────────┘     │
│                                                             │
│   흩어진 데이터를      도구가 엮어서       인사이트가 완성된다    │
│   한 곳에 모은다       의미를 만든다                            │
└─────────────────────────────────────────────────────────────┘
```

| 세계관 | 역할 | 실체 |
|--------|------|------|
| **Thread** | 워크스페이스 | 광고주 단위의 작업 공간. 데이터 격리 기준 |
| **Fiber** | 데이터소스 | 소셜 버즈, 검색량, 감성 신호 등 원재료 |
| **Loom** | 도구 | AIM, 소셜 클러스터링 등 분석 엔진 |
| **Ribbon** | 산출물 | 점수, 차트, 자연어 해석이 담긴 완성된 결과물 |

## 3. 사용자 여정 (User Journey)

### 3.1 기본 흐름

```
① 로그인 → 홈 진입
     │
② 내 Thread 목록 확인 (= 내가 권한 가진 광고주)
     │
③ Thread 선택 (예: "삼성전자")
     │
④ Fiber 확인: 이 Thread에 어떤 데이터가 모여있는가?
     │
⑤ Loom 선택: 어떤 분석을 돌릴 것인가?
     │
     ├─ 데이터 충족 확인: 이 Loom에 필요한 Fiber가 충분한가?
     │   ├─ ✅ 충족 → 실행
     │   └─ ❌ 미충족 → "이 Fiber를 먼저 수집하세요" 안내
     │
⑥ Loom 실행 → Ribbon 생성
     │
⑦ Ribbon 열람: 점수 + 차트 + HEXIM 자연어 해석
     │
⑧ (선택) Ribbon 공유 / PDF 내보내기 / 알림 설정
```

### 3.2 대화형 숏컷 (AI 어시스트 모드)

```
사용자: "이번 달 갤럭시 S25 AIM 뽑아줘"
     │
AI 자동 처리:
  ├─ 키워드 추출: "갤럭시 S25"
  ├─ 기간 추론: 이번 달 (2026.06)
  ├─ Fiber 확인: 소셜 버즈 ✅, 검색량 ✅, 감성 ✅
  ├─ Loom 선택: AIM
  │
  ▼
Ribbon 자동 생성
```

## 4. Fiber (데이터소스) 구조

### 4.1 추상화 원칙

사용자에게는 실제 소스명(트렌드업, 리스닝마인드)을 노출하지 않는다. **데이터 유형명**만 보여준다.

### 4.2 Fiber 목록

| Fiber명 | 데이터 내용 | 사용자 입력 |
|---------|-----------|------------|
| 소셜 버즈 | 언급 건수 추이, 채널별 분포 | 키워드 + 기간 |
| 오디언스 보이스 | 소셜 언급 원문 텍스트 | 키워드 + 기간 |
| 감성 신호 | 긍정/부정/중립 비율 추이 | 키워드 + 기간 |
| 맥락 키워드 | TPO, 연관 키워드 | 키워드 |
| 검색량 | 검색 볼륨 시계열 | 키워드 + 기간 |
| 연관 검색어 | 함께 검색되는 키워드, 관련 주제 | 키워드 |
| 검색 의도 | 인텐트 분류, 검색 여정 | 키워드 |
| X 언급 (via Grok) | X 멘션 요약, 감성, 주요 트윗 | 키워드 + 기간 |
| 캠페인 브리프 | 기획 맥락, 타겟 가설 | 파일 업로드 |
| 외부 레퍼런스 | 뉴스, 아티클, 업계 동향 | 키워드 (자동) |
| 마케팅 유입 | MAU, 세션 수 | 파일 업로드 or GA 연동 |

### 4.3 Fiber 수집 모드

| 모드 | 설명 |
|------|------|
| **자동 수집** | 키워드+기간 입력하면 백엔드가 알아서 가져옴 |
| **수동 업로드** | CSV/엑셀 파일 직접 업로드 (캠페인 브리프, GA 데이터 등) |
| **대화형 추출** | 챗에서 의도를 말하면 AI가 키워드·기간·Fiber 유형 자동 설정 |

## 5. Loom (도구) 구조

### 5.1 Loom의 동작 원리

```
[사용자가 Loom 선택]
     │
     ▼
[데이터 충족 검증]
  "이 Loom에 필요한 Fiber가 Thread에 있는가?"
     │
     ├─ ✅ 충족 → [Loom 실행]
     │              │
     │              ▼
     │         [HEXIM 해석 레이어]
     │              │
     │              ▼
     │         [Ribbon 생성]
     │
     └─ ❌ 미충족 → [부족한 Fiber 안내 + 수집 유도]
```

### 5.2 Phase 1 Loom

| Loom | 기능 | 필수 Fiber | 산출물 (Ribbon) |
|------|------|-----------|----------------|
| **AIM** | 브랜드 종합 지표 (50점 기준) | 소셜 버즈, 검색량, 감성 신호 | 종합/단기 AIM 점수 + 시계열 차트 + 해석 |
| **소셜 클러스터링** | 오디언스 반응 유형 분류 | 오디언스 보이스 | 클러스터별 키워드 맵 + 원문 샘플 + 요약 |

### 5.3 Phase 2 Loom (예정)

| Loom | 기능 | 필수 Fiber |
|------|------|-----------|
| 경쟁사 비교 | 자사 vs 경쟁사 AIM/버즈 비교 | 소셜 버즈 (복수 브랜드) |
| 캠페인 효과 측정 | 사전/사후 비교 | 소셜 버즈, 감성, 캠페인 브리프 |
| 영상 분석 | 영상 콘텐츠 반응 분석 | (별도 정의 예정) |

## 6. Ribbon (산출물) 구조

### 6.1 Ribbon의 구성 요소

모든 Ribbon은 아래 3가지를 기본으로 포함한다:

```
┌─────────────────────────────────────────┐
│              Ribbon                      │
├─────────────────────────────────────────┤
│  📊 시각화 (차트, 그래프, 맵)            │
│     • AIM 시계열 차트                    │
│     • 클러스터 시각화                    │
│     • 개별 지표 드릴다운                 │
├─────────────────────────────────────────┤
│  📝 자연어 해석 (HEXIM)                  │
│     • "전월 대비 3점 상승,               │
│       주요 원인은 인스타 버즈 급증"       │
│     • 핵심 내러티브 요약                 │
├─────────────────────────────────────────┤
│  📋 원본 데이터 (드릴다운)               │
│     • 수치 테이블                        │
│     • 원문 샘플 (오디언스 보이스)         │
└─────────────────────────────────────────┘
```

### 6.2 Ribbon 생명주기

```
[생성] → [열람] → [공유/내보내기] → [아카이브]
                                      │
                     [자동 갱신] ←─────┘ (Saga 모드 설정 시)
```

- **자동 갱신**: Fiber가 새로 수집되면 Ribbon을 자동 재생성 (선택 설정)
- **버저닝**: 같은 Loom을 다른 시점에 돌리면 이전 Ribbon과 비교 가능

## 7. HEXIM (해석 엔진) — Reborn의 핵심 차별화

### 7.1 왜 HEXIM이 핵심인가

| 기존 방식 (Brandwatch, Pulsar 등) | Reborn 방식 (HEXIM) |
|----------------------------------|---------------------|
| 데이터/차트를 보여주고 해석은 사용자 몫 | **해석을 먼저 생성하고**, 근거 데이터를 함께 제공 |
| 전문가가 없으면 인사이트 도출 불가 | 비전문가도 즉시 보고서에 활용 가능 |
| "Ask"해야 답을 줌 | **질문 없이도** 선제적으로 설명 |

### 7.2 HEXIM 동작 방식

```
[Loom 실행 결과 (점수, 클러스터 등)]
     │
     ▼
[맥락 데이터 수집]
  • 전월 대비 변화량
  • 가장 큰 변동 요인 (어떤 Fiber가 얼마나 변했나)
  • 관련 이벤트/뉴스 (외부 레퍼런스)
     │
     ▼
[프롬프트 구성 → LLM 호출]
     │
     ▼
[자연어 해석 텍스트 생성]
  예: "6월 AIM 53점 (+3). 인스타그램 버즈가 전월 대비 42% 증가하며
      전체 점수를 끌어올렸습니다. 주요 내러티브는 '여름 신제품 기대감'으로,
      20~30대 여성 중심으로 확산되고 있습니다."
```

## 8. 확장 모델: 새 Loom 추가 시

Reborn의 파이프라인 구조는 새로운 도구(Loom)를 빠르게 플러그인할 수 있도록 설계된다.

### 새 Loom 추가에 필요한 것

| 단계 | 정의할 것 | 예시 (경쟁사 비교 Loom) |
|------|----------|----------------------|
| 1 | 필요한 Fiber 정의 | 소셜 버즈 (자사 + 경쟁사 키워드) |
| 2 | 분석 로직 정의 | 양 브랜드 AIM 산출 → 차이 비교 |
| 3 | 산출물(Ribbon) 형태 정의 | 비교 차트 + 강점/약점 해석 |
| 4 | HEXIM 해석 템플릿 작성 | "자사는 검색량에서 우위, 경쟁사는 감성에서 우위..." |
| 5 | UI 컴포넌트 개발 | 비교 대시보드 위젯 |

→ **기존 Fiber와 HEXIM 인프라를 재활용**하므로, 새 Loom은 분석 로직 + UI만 추가하면 됨.

## 9. ReadySet ↔ Reborn 연동 구조

```
┌──────────────┐                    ┌──────────────┐
│   ReadySet   │                    │    Reborn    │
│  (사전 전략)  │                    │  (사후 검증)  │
├──────────────┤                    ├──────────────┤
│ 타겟 가설    │ ──── 브리프 ────▶  │ 캠페인 브리프 │
│ 도달 예측    │      전달          │  (Fiber)     │
│              │                    │              │
│              │  ◀── AIM 결과 ──── │ AIM Ribbon   │
│ 차기 전략    │      환류          │ (인사이트)    │
│ 수정 근거    │                    │              │

인사이트 피드백 루프:
ReadySet 가설 → 캠페인 집행 → Reborn 검증 → 차기 ReadySet 전략 수정
```

## 11. 거버넌스 & 접근 제어

### 11.1 유저 롤별 권한

| 행위 | Admin | Marketer | Agency | Client |
|------|:---:|:---:|:---:|:---:|
| Thread 생성 | ✅ | ✅ | ❌ | ❌ |
| Fiber 수집 설정 | ✅ | ✅ | ✅ | ❌ |
| Loom 실행 | ✅ | ✅ | ✅ | ❌ |
| Ribbon 열람 | ✅ | ✅ | ✅ | ✅ |
| Ribbon 공유/내보내기 | ✅ | ✅ | ✅ | ✅ |
| 광고주 권한 관리 | ✅ | ❌ | ❌ | ❌ |
| 시스템 설정 | ✅ | ❌ | ❌ | ❌ |

### 11.2 데이터 격리

```
유저 로그인
  → 보유 광고주 권한 조회
  → 해당 광고주의 Thread만 노출
  → Thread 내 모든 Fiber/Ribbon은 해당 광고주 데이터만 포함
```

- 1 유저 → N개 광고주 권한 가능
- 광고주 A의 데이터가 광고주 B Thread에 절대 노출되지 않음

## 12. 핵심 설계 원칙 요약

| 원칙 | 설명 |
|------|------|
| **파이프라인이 건물** | Fiber → Loom → Ribbon 구조가 먼저. 개별 도구는 이 위에 올라가는 방 |
| **해석이 기본값** | 모든 Ribbon에 HEXIM 자연어 해석 포함. 숫자만 던지지 않음 |
