# Reborn — 솔루션 아키텍처

> 전략/기획 관점에서 Reborn 플랫폼의 구조, 흐름, 확장 방향을 정의한 문서

---

## 1. 핵심 구조: 직조 파이프라인

Reborn은 **"Fiber → Weave → Fabric → Re:born"** 의 직조 파이프라인을 기본 골격으로 한다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                       Thread (워크스페이스)                           │
│                       = 광고주 단위 작업 공간                         │
│                                                                     │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐ │
│   │  Fiber   │ ──▶ │  Weave   │ ──▶ │  Fabric  │ ──▶ │ Re:born  │ │
│   │ (원료)   │     │ (엮다)   │     │ (직물)   │     │ (페이지) │ │
│   └──────────┘     └──────────┘     └──────────┘     └──────────┘ │
│                                                                     │
│   흩어진 데이터를    도구가 엮어서     개별 결과물이     Fabric들이     │
│   한 곳에 모은다    의미를 만든다     생성된다          한 페이지에     │
│                                                       모여 인사이트로 │
│                                                       Re:born 된다   │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.1 세계관 정의

| 세계관 | 역할 | 정의 |
|--------|------|------|
| **Thread** | 워크스페이스 | 하나의 브랜드·캠페인을 분석하기 위한 독립된 작업 공간. 광고주 단위로 데이터가 격리되며, 모든 Fiber·Weave·Fabric이 이 안에 소속된다. |
| **Fiber** | 데이터소스 | Thread에 공급되는 원천 데이터 한 묶음. 외부 소스에서 수집되어 정규화된 상태로 Thread에 적재된다. 하나의 Thread에 여러 종류의 Fiber가 존재한다. |
| **Weave** | 도구 | Fiber를 입력받아 특정 분석 로직을 수행하는 도구. 실행하면 하나의 Fabric을 생성한다. 각 Weave는 실행에 필요한 Fiber 조건이 사전 정의되어 있다. |
| **Fabric** | 개별 산출물 | Weave를 1회 실행하여 생성된 분석 결과물. 시각화(차트) + 자연어 해석(HEXIM/SpinX) + 원본 데이터 드릴다운으로 구성된다. |
| **Re:born** | 최종 산출물 | 하나 이상의 Fabric이 모여있는 페이지 형태의 종합 인사이트. 데이터가 인사이트로 다시 태어난(Re:born) 결과물. 광고주에게 전달되는 최종 형태. |

### 1.2 AI 레이어

| 이름 | 역할 | 정의 |
|------|------|------|
| **HEXIM** | 생성형 AI 엔진 | Reborn 플랫폼의 생성형 AI 엔진. 모든 해석·요약·분석 자동화의 기반 기술. |
| **SpinX** | AI Agent | HEXIM 엔진 기반으로 동작하는 AI 에이전트. 대화형 숏컷(AI 어시스트)과 Fabric 생성 과정에 개입하여 사용자를 보조한다. |

## 3. Thread 기본 설정값

Thread 생성 시 아래 정보를 설정한다. 이 값들이 이후 Fiber 수집과 Weave 실행의 공통 조건으로 사용된다.

### 3.1 필수 설정

| 필드 | 설명 | 예시 |
|------|------|------|
| Thread명 | 워크스페이스 이름 | "삼성 갤럭시 S25" |
| 광고주 맵핑 | 소속 광고주 선택 (데이터 격리 기준) | 삼성전자 |
| 가시성 | 접근 범위 설정 | Private / Internal / Shared |
| 대표 브랜드명 | 분석 대상 브랜드 | "갤럭시 S25" |
| 핵심 키워드 | 데이터 수집에 사용할 키워드 (복수) | "갤럭시", "S25", "삼성 스마트폰" |
| 분석 기간 | 데이터 수집·분석 기본 기간 | 2025.07 ~ 2026.06 (최근 12개월) |

### 3.2 선택 설정

| 필드 | 설명 | 예시 |
|------|------|------|
| 경쟁사 | 비교 대상 브랜드/키워드 | "아이폰16", "픽셀9" |
| 경쟁사 키워드 | 경쟁사별 수집 키워드 | "아이폰", "iPhone 16" |

### 3.3 가시성 정책 (ReadySet 동일)

| 레벨 | 설명 |
|------|------|
| **Private** | 생성자 본인만 접근 |
| **Internal** | 같은 팀(같은 광고주 권한 보유자) 전체 접근 |
| **Shared** | 광고주(Client 롤)에게도 열람 공유 |

### 3.4 Thread 설정값의 역할

```
Thread 설정 (키워드, 기간, 경쟁사)
     │
     ├─→ Fiber 수집 시 기본 조건으로 자동 적용
     │     (매번 키워드·기간 재입력 불필요)
     │
     ├─→ Weave 실행 시 기본 파라미터로 참조
     │
     └─→ SpinX가 대화형 요청 시 자동 참조
           ("AIM 뽑아줘" → Thread 키워드·기간 자동 사용)
```

## 4. 사용자 여정 (User Journey)

### 4.1 기본 흐름

```
① 로그인 → 홈 진입
     │
② 내 Thread 목록 확인 (= 내가 권한 가진 광고주)
     │
③ Thread 선택 (예: "삼성 갤럭시 S25")
     │
④ Fiber 확인: 이 Thread에 어떤 데이터가 모여있는가?
     │
⑤ Weave 선택: 어떤 분석을 돌릴 것인가?
     │
     ├─ 데이터 충족 확인: 이 Weave에 필요한 Fiber가 충분한가?
     │   ├─ ✅ 충족 → 실행
     │   └─ ❌ 미충족 → "이 Fiber를 먼저 수집하세요" 안내
     │
⑥ Weave 실행 → SpinX가 Fabric 생성
     │
⑦ Fabric 열람: 점수 + 차트 + 자연어 해석
     │
⑧ (선택) 여러 Fabric을 Re:born 페이지에 구성 → 광고주에게 전달
```

### 4.2 대화형 숏컷 (SpinX 모드)

```
사용자: "이번 달 갤럭시 S25 AIM 뽑아줘"
     │
SpinX 자동 처리:
  ├─ Thread 설정값에서 키워드/기간 자동 참조
  ├─ Fiber 확인: 소셜 버즈 ✅, 검색량 ✅, 감성 ✅
  ├─ Weave 선택: AIM
  │
  ▼
Fabric 자동 생성 (AIM 점수 + 차트 + 해석)
```

## 5. Fiber (데이터소스) 구조

### 5.1 추상화 원칙

사용자에게는 실제 소스명(트렌드업, 리스닝마인드)을 노출하지 않는다. **데이터 유형명**만 보여준다.

### 5.2 Fiber 목록

| Fiber명 | 데이터 내용 | 사용자 입력 |
|---------|-----------|------------|
| 소셜 버즈 | 언급 건수 추이, 채널별 분포 | 키워드 + 기간 |
| 오디언스 보이스 | 소셜 언급 원문 텍스트 | 키워드 + 기간 |
| 감성 신호 | 긍정/부정/중립 비율 추이 | 키워드 + 기간 |
| 맥락 키워드 | TPO, 연관 키워드 | 키워드 |
| 검색량 | 검색 볼륨 시계열 | 키워드 + 기간 |
| 연관 검색어 | 함께 검색되는 키워드, 관련 주제 | 키워드 |
| 검색 의도 | 인텐트 분류, 검색 여정 | 키워드 |
| X 언급 (via Grok) | X 멘션 요약, 감성, 주요 트윗 | 키워드 + 기간 |
| 캠페인 브리프 | 기획 맥락, 타겟 가설 | 파일 업로드 |
| 외부 레퍼런스 | 뉴스, 아티클, 업계 동향 | 키워드 (자동) |
| 마케팅 유입 | MAU, 세션 수 | 파일 업로드 or GA 연동 |

### 5.3 Fiber 수집 모드

| 모드 | 설명 |
|------|------|
| **자동 수집** | 키워드+기간 입력하면 백엔드가 알아서 가져옴 |
| **수동 업로드** | CSV/엑셀 파일 직접 업로드 (캠페인 브리프, GA 데이터 등) |
| **SpinX 추출** | 챗에서 의도를 말하면 SpinX가 키워드·기간·Fiber 유형 자동 설정 후 수집 |

## 6. Weave (도구) 구조

### 6.1 Weave의 동작 원리

```
[사용자가 Weave 선택 또는 SpinX가 자동 선택]
     │
     ▼
[데이터 충족 검증]
  "이 Weave에 필요한 Fiber가 Thread에 있는가?"
     │
     ├─ ✅ 충족 → [Weave 실행]
     │              │
     │              ▼
     │         [SpinX가 HEXIM 엔진으로 해석 생성]
     │              │
     │              ▼
     │         [Fabric 생성]
     │
     └─ ❌ 미충족 → [부족한 Fiber 안내 + 수집 유도]
```

### 6.2 Phase 1 Weave

| Weave | 기능 | 필수 Fiber | 산출물 (Fabric) |
|------|------|-----------|----------------|
| **AIM** | 브랜드 종합 지표 산출 (50점 기준). 검색량·버즈·감성을 복합 가중하여 브랜드 상태 변화를 단일 점수로 요약 | 소셜 버즈, 검색량, 감성 신호 | 종합/단기 AIM 점수 + 시계열 차트 + 해석 |
| **CEP 인사이트** | 소셜 원문에서 Category Entry Point(소비자가 해당 카테고리에 진입하는 상황·맥락)를 추출·분류 | 오디언스 보이스 | CEP 클러스터 분류 + 맥락별 빈도 + 해석 |
| **쿼리 데이터 분석** | 검색 키워드 데이터에서 소비자 의도·트렌드·연관 관심사를 분석 | 검색량, 연관 검색어, 검색 의도 | 검색 트렌드 차트 + 의도 분류 + 해석 |
| **X 반응 분석** ⚡ | X(트위터) 실시간 반응을 요약·분류하여 브랜드 여론 감지 | X 언급 (via Grok) | 주요 반응 요약 + 감성 분포 + 핵심 트윗 |

> ⚡ X 반응 분석: 가능하면 Phase 1 포함 (Grok API 안정성 검증 후 확정)
> ⚠️ CEP 인사이트 원천: 트렌드업 외 추가 소스 검토 중
> ⚠️ 쿼리 데이터 분석 원천: 데이터 소스 검토 중

### 6.3 Phase 2 Weave (예정)

| Weave | 기능 | 필수 Fiber |
|------|------|-----------|
| 경쟁사 비교 | 자사 vs 경쟁사 AIM/버즈 비교 | 소셜 버즈 (복수 브랜드) |
| 캠페인 효과 측정 | 캠페인 사전·사후 지표 비교 | 소셜 버즈, 감성 신호, 캠페인 브리프 |
| 영상 분석 | 영상 콘텐츠 반응 분석 | (별도 정의 예정) |

## 7. Fabric & Re:born (산출물) 구조

### 7.1 Fabric의 구성 요소

모든 Fabric은 아래 3가지를 기본으로 포함한다:

```
┌─────────────────────────────────────────┐
│              Fabric                      │
├─────────────────────────────────────────┤
│  📊 시각화 (차트, 그래프, 맵)            │
│     • AIM 시계열 차트                    │
│     • 클러스터 시각화                    │
│     • 개별 지표 드릴다운                 │
├─────────────────────────────────────────┤
│  📝 자연어 해석 (SpinX / HEXIM)          │
│     • "전월 대비 3점 상승,               │
│       주요 원인은 인스타 버즈 급증"       │
│     • 핵심 내러티브 요약                 │
├─────────────────────────────────────────┤
│  📋 원본 데이터 (드릴다운)               │
│     • 수치 테이블                        │
│     • 원문 샘플 (오디언스 보이스)         │
└─────────────────────────────────────────┘
```

### 7.2 Re:born 페이지

Re:born은 Fabric들이 모여있는 **페이지** 형태이다.

```
┌─────────────────────────────────────────────────┐
│          Re:born — "6월 삼성전자"                 │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌───────────────────────────────────────┐      │
│  │  Fabric 1: AIM                        │      │
│  │  종합 53점 (+3) | 단기 55점 (+5)       │      │
│  │  "인스타 버즈 42% 증가가 주요 원인..."  │      │
│  └───────────────────────────────────────┘      │
│                                                 │
│  ┌───────────────────────────────────────┐      │
│  │  Fabric 2: CEP 인사이트                │      │
│  │  주요 진입 맥락 4개 클러스터 도출       │      │
│  │  "여름 여행 준비 상황에서 진입 증가..." │      │
│  └───────────────────────────────────────┘      │
│                                                 │
│  ┌───────────────────────────────────────┐      │
│  │  Fabric 3: X 반응 분석                 │      │
│  │  감성: 긍정 62% | 주요 키워드: ...     │      │
│  └───────────────────────────────────────┘      │
│                                                 │
│  [PDF 내보내기]  [공유 링크 생성]                 │
└─────────────────────────────────────────────────┘
```

### 7.3 Fabric 생명주기

```
[Weave 실행] → [Fabric 생성] → [Re:born 페이지에 배치] → [열람/공유/내보내기]
```

- Fabric은 **수동으로 Weave를 실행할 때만 생성**됨 (자동 갱신 없음)
- 같은 Weave를 다른 시점에 돌리면 새 Fabric이 생성됨 → 이전 결과와 비교 가능 (버저닝)
- 불필요한 Fabric은 아카이브/삭제 가능

## 8. HEXIM & SpinX — AI 레이어

### 8.1 HEXIM (생성형 AI 엔진)

| 항목 | 내용 |
|------|------|
| 정의 | Reborn 플랫폼의 생성형 AI 엔진. 모든 자연어 해석·요약·분류 자동화의 기반 기술 |
| 역할 | Weave 결과를 받아 자연어 해석 텍스트를 생성하는 핵심 엔진 |
| 적용 범위 | 모든 Fabric의 해석 레이어 + SpinX 에이전트의 동력원 |

### 8.2 SpinX (AI Agent)

| 항목 | 내용 |
|------|------|
| 정의 | HEXIM 엔진 기반으로 동작하는 AI 에이전트 |
| 역할 | 사용자와 대화하며 Fiber 수집·Weave 실행·Fabric 생성을 자동화 |
| 개입 지점 | ① 대화형 숏컷 (의도 파악 → 자동 실행) ② Fabric 생성 시 해석 자동 작성 |
| 이름 유래 | Spin(실을 잣다) + X(미지수, 가능성) — 데이터를 돌려서 인사이트를 뽑아내는 에이전트 |

### 8.3 SpinX 동작 방식

```
[사용자 입력 또는 Weave 실행 결과]
     │
     ▼
[SpinX — HEXIM 엔진 호출]
  • 맥락 데이터 수집 (전월 비교, 변동 요인, 관련 이벤트)
  • 프롬프트 구성
  • LLM 호출
     │
     ▼
[자연어 해석 텍스트 생성]
  예: "6월 AIM 53점 (+3). 인스타그램 버즈가 전월 대비 42% 증가하며
      전체 점수를 끌어올렸습니다. 주요 내러티브는 '여름 신제품 기대감'으로,
      20~30대 여성 중심으로 확산되고 있습니다."
```

### 8.4 왜 HEXIM/SpinX가 핵심 차별화인가

| 기존 방식 (Brandwatch, Pulsar 등) | Reborn 방식 (HEXIM/SpinX) |
|----------------------------------|--------------------------|
| 데이터/차트를 보여주고 해석은 사용자 몫 | **해석을 먼저 생성하고**, 근거 데이터를 함께 제공 |
| 전문가가 없으면 인사이트 도출 불가 | 비전문가도 즉시 보고서에 활용 가능 |
| "Ask"해야 답을 줌 | **질문 없이도** 선제적으로 설명 |
| AI가 보조 역할 | SpinX가 **전체 플로우를 주도** 가능 |

## 9. 확장 모델: 새 Weave 추가 시

Reborn의 파이프라인 구조는 새로운 도구(Weave)를 빠르게 플러그인할 수 있도록 설계된다.

### 새 Weave 추가에 필요한 것

| 단계 | 정의할 것 | 예시 (경쟁사 비교 Weave) |
|------|----------|----------------------|
| 1 | 필요한 Fiber 정의 | 소셜 버즈 (자사 + 경쟁사 키워드) |
| 2 | 분석 로직 정의 | 양 브랜드 AIM 산출 → 차이 비교 |
| 3 | 산출물(Fabric) 형태 정의 | 비교 차트 + 강점/약점 해석 |
| 4 | SpinX 해석 템플릿 작성 | "자사는 검색량에서 우위, 경쟁사는 감성에서 우위..." |
| 5 | UI 컴포넌트 개발 | 비교 대시보드 위젯 |

→ **기존 Fiber와 HEXIM/SpinX 인프라를 재활용**하므로, 새 Weave는 분석 로직 + UI만 추가하면 됨.

## 12. 거버넌스 & 접근 제어

### 12.1 유저 롤별 권한

| 행위 | Admin | Marketer | Agency | Client |
|------|:---:|:---:|:---:|:---:|
| Thread 생성/수정/삭제 | ✅ | ✅ | ❌ | ❌ |
| Fiber 수집 설정 | ✅ | ✅ | ❌ | ❌ |
| Weave 실행 | ✅ | ✅ | ❌ | ❌ |
| Fabric·Re:born 열람 (R) | ✅ | ✅ | ✅ | ✅ |
| Fabric·Re:born 공유/내보내기 | ✅ | ✅ | ✅ | ✅ |
| 광고주 권한 관리 | ✅ | ❌ | ❌ | ❌ |
| 시스템 설정 | ✅ | ❌ | ❌ | ❌ |

> Agency, Client = **Read Only**. 생성·수정·삭제·실행 권한 없음. 결과물 열람 및 공유만 가능.

### 12.2 데이터 격리

```
유저 로그인
  → 보유 광고주 권한 조회
  → 해당 광고주의 Thread만 노출
  → Thread 내 모든 Fiber/Fabric은 해당 광고주 데이터만 포함
```

- 1 유저 → N개 광고주 권한 가능
- 광고주 A의 데이터가 광고주 B Thread에 절대 노출되지 않음

## 13. 핵심 설계 원칙 요약

| 원칙 | 설명 |
|------|------|
| **파이프라인이 건물** | Fiber → Weave → Fabric → Re:born 구조가 먼저. 개별 Weave는 이 위에 올라가는 방 |
| **해석이 기본값** | 모든 Fabric에 SpinX(HEXIM) 자연어 해석 포함. 숫자만 던지지 않음 |
| **대화로 시작** | SpinX가 진입 장벽 제거. 키워드·기간·Fiber·Weave를 자동 설정 |
| **소스 추상화** | 사용자는 "소셜 버즈"를 보지, "트렌드업"을 보지 않음 |
| **플러그인 확장** | 새 Weave = 새 분석 로직 + UI만 추가. 파이프라인 구조 변경 없음 |
| **광고주 격리** | 데이터 격리의 최소 단위 = 광고주. Thread가 이를 실현 |
| **수동 갱신** | Fabric은 사용자가 Weave를 실행할 때만 생성. 자동 갱신 없음 |
